Git Command Cheat Sheet

Tu guía rápida de comandos esenciales aprendidos en el bootcamp.

1. Configuración Inicial

Lo primero que haces en una PC nueva. Fundamental para que tus commits tengan tu firma.

# Configura tu identidad global git config --global user.name "Tu Nombre" git config --global user.email "tu@email.com" # Ajustes de calidad de vida recomendados git config --global init.defaultBranch main

2. Arrancando Motores

Cómo empezar un proyecto, ya sea desde cero en tu PC o trayéndolo de la nube.

# Convierte la carpeta actual en un repositorio git init # Descarga un proyecto existente (con toda su historia) git clone [URL_SSH_DEL_REPO]

3. El Ciclo Diario

El flujo sagrado de trabajo: trabajar, preparar y guardar. Repetir 100 veces al día.

# Comprueba qué ha cambiado (ROJO = sin preparar, VERDE = listo) git status # Prepara TODOS los archivos cambiados para la foto (Stage) git add . # Toma la foto y guarda la versión con un mensaje git commit -m "Verbo imperativo describiendo el cambio"

4. Ramas (Universos Paralelos)

Nunca trabajes directo en `main`. Crea espacios seguros para tus experimentos.

# Crea UNA NUEVA rama y te mueve a ella automáticamente git checkout -b nombre-de-la-nueva-rama # Regresa a la rama principal (sin crear nada nuevo) git checkout main

5. Nube y Sincronización

Conectando tu PC local con GitHub. Subir tus cambios y bajar los de tus compañeros.

# Sube tu rama actual a GitHub por primera vez (publicar) git push -u origin nombre-de-tu-rama # Baja los cambios nuevos de la nube a tu rama actual git pull origin main

6. Información e Historial

Comandos para no perderse y ver qué ha pasado antes.

# Muestra el historial de commits resumido en una línea git log --oneline # Verifica la conexión SSH con GitHub ssh -T git@github.com
